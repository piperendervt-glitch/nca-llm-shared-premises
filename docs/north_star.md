## A-4の精密版（MVE-01〜07で確立・2026-04-05確定）

```
精密版:
「shared_premisesの欠如は
 異なる固有前提H*_iを持つ
 エージェントが混在した場面で顕在化する

 単一モデルは自分のH*_iに従って
 決定論的に動作するが（flip rate≈0%・MVE-07で確認）
 H*_iが真のH*からずれている場合
 タイプC（前提が間違っている）の失敗が生じる

 異種モデル混在では
 H*_iのズレが可視化され
 split率として観測される」

形式定義との接続（ICLR 2025・Iyer et al.）:
  タスク仕様 S = (R, C)
  C = shared_premises
  各モデルiは固有の前提H*_iを持つ
  H*_iのズレ = split率として観測

適用条件（MVE-01〜06で実証）:
  判断が曖昧なタスク（H*_iのズレが顕在化しやすい）
    = world_consistency・contradiction・compound
    → 多様性効果が有効（split_acc差: +17〜+48pp）

  答えが決定論的なタスク（H*_iが収束しやすい）
    = syllogism・transitivity・math_middle/high
    → 多様性効果が逆効果（split_acc差: -46〜-100pp）
```

## 曖昧さの測定指標（確定・2026-04-05）

```
主指標:
  split率: n_split / n_total
    = 「3ノード間でH*_iのズレが顕在化した割合」
    = そのモデルアンサンブルにとっての曖昧さ

  calibration error: |mean_confidence - accuracy|
    = 「モデルが過信している度合い」
    = A-4のタイプC（前提が間違っている）と接続

  両指標の独立性: r=0.293（確認済み）
  cal_error vs CFR: r=0.728（強い相関）

補助指標:
  CFRベースの曖昧さスコア（オンライン更新）:
    ambiguity(τ)(t+1) =
      (1-α)·ambiguity(τ)(t) + α·1[unanimous(t)∧¬correct(t)]
    初期値: MVE-01〜06のCFRで初期化
    外堀実験を進めながらオンライン更新

除外した指標（理由とともに記録）:
  flip rate（MVE-07で除外確定）:
    qwen2.5:7bはtemperature=0.8でも決定論的
    全タスクで0〜3.2%（識別力なし）
    発見: 「曖昧さはモデル内部の不安定さではなく
           モデル間の前提差異として顕在化する」

  LLM直接自己申告（confidence）:
    verbalized confidenceのΔ=0.004〜0.021
    識別力がほぼゼロ（MVE-01〜06で確認）

  情報エントロピー（vote_distributionベース）:
    split率と数学的に等価（独立指標にならない）

  Level-K（条件横断の正解率ばらつき）:
    accuracyと強い逆相関（r < -0.8と予想）
    accuracyの別表現になる可能性が高い
    粒度の不一致（タスク単位 vs 条件単位）
```

## 外堀実験B〜EのSuccess Criteria（事前宣言・変更禁止）

```
Must（主指標）:
  CFR差（7b_het − 7b_homo）> +5pp
  かつ方向が正（7b_hetの方がCFRが低い）

根拠:
  +5ppはMVE-01〜06の最小有意観測差から設定
  world_consistencyのCFR差=16.3ppが最大観測値
  +5ppは保守的かつ妥当な閾値

補助指標:
  split率とCFR差のSpearman相関 r > 0.5
  曖昧タスク群（split率上位50%）でCFR差が特に大きい

外堀実験の順序（事前宣言）:
  B: 常識推論（曖昧さ大・多様性有効の予想）
  D: 因果推論（曖昧さ大・多様性有効の予想）
  E: 文法・言語（曖昧さ中・混在の予想）
  C: コード判定（曖昧さ小・多様性逆効果の予想）
```
