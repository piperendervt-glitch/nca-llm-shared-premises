## ④ Success Criteria（成功基準）

| 指標 | ベースライン | 成功閾値 | 優先度 |
|------|------------|---------|--------|
| 全体エラー率（NCA vs 単一） | 未測定 | NCA > 単一エージェント3モデル平均 | 必須 |
| CFR（NCA-LLMのみ） | 未測定 | > 20% | 必須 |
| 全員一致率（NCA-LLM） | 未測定 | > 50% | 参考 |

**判定ロジック:**
```
必須指標1: 全体エラー率の比較（同じ定義で比較）
  NCA-LLM エラー率 > 単一エージェント平均エラー率
  → SUCCESS: グループシンクで精度が下がる
  → FAIL: NCA構造はグループシンクを引き起こさない

必須指標2: CFR（NCA-LLMのみ）
  wrong_unanimous / total_unanimous > 20%
  → SUCCESS: 全員一致した場合に間違いが頻発する
  → FAIL: 全員一致は信頼できる信号として機能している
```

**各指標の定義（必ず生データから計算）:**
```
全体エラー率（単一・NCA共通）:
  error_rate = n_wrong / n_total
  ※同じ定義で単一エージェントとNCA-LLMを比較できる

CFR（NCA-LLMのみ）:
  CFR = wrong_unanimous / total_unanimous
  wrong_unanimous: 全ノード一致かつ間違いだったタスク数
  total_unanimous: 全ノードが同じ答えを出したタスク数
  ※単一エージェントには適用しない（定義が非対称になるため）

全員一致率（NCA-LLMのみ）:
  unanimity_rate = total_unanimous / n_total
```

## ⑤ Prediction（事前予想）

| 指標 | 予想値 | 根拠 |
|------|--------|------|
| NCA-LLM CFR | 25〜40% | v1の精度が45〜55%程度だったため |
| 単一エージェントCFR（qwen） | 30〜50% | 単独では引きずられにくい |
| 単一エージェントCFR（llama3） | 30〜50% | 同上 |
| 単一エージェントCFR（mistral） | 30〜50% | 同上 |
| 全員一致率（NCA） | 70〜90% | NCAの相互参照で収束しやすい |

**予想の確信度:** 低

理由:
  v1の旧データはあるが
  CFRを定義通りに計算した記録がない
  新しく測定する必要がある

**予想が外れた場合に最初に疑うこと:**
```
Case A: NCA CFR < 単一エージェントCFR
  → NCA構造が「誤りの修正」に機能している
  → Claimが棄却される
  → 「NCA構造はグループシンクではなく
    エラー修正をしている」として記録

Case B: 単一エージェントCFRが非常に高い（>50%）
  → world_consistencyタスク自体が難しすぎる
  → ベースラインが高すぎてNCAとの差が出ない
  → タスクの難易度調整が必要かもしれない

Case C: 全員一致率が低い（<30%）
  → NCA構造で意見が分かれている
  → グループシンクではなく「多様性」が起きている
  → Claimの前提（全員一致が多発する）が崩れる
```

## 実験結果

> 実験完了: 2026-04-04

| 指標 | 予想値 | 実際の値 | 差分 |
|------|--------|---------|------|
| NCA エラー率 | 44〜48% | 28.0% | -16〜-20pp |
| 単一エラー率（qwen2.5:7b） | 30〜50% | 41.0% | — |
| 単一エラー率（llama3） | 30〜50% | 39.0% | — |
| 単一エラー率（mistral:7b） | 30〜50% | 33.0% | — |
| 単一エラー率（平均） | 30〜50% | 37.7% | — |
| CFR（NCA-LLMのみ） | 25〜40% | 14.3% [4.8%, 30.3%] | -10〜-26pp |
| 全員一致率（NCA） | 70〜90% | 35.0% | -35〜-55pp |

**達成判定:** FAIL

```
必須指標1: NCA エラー率 > 単一エージェント平均
  28.0% < 37.7%（NCAの方が精度が高い・逆方向）
  delta = -9.7pp, p = 0.08
  → FAIL ❌

必須指標2: CFR > 20%
  14.3% < 20%
  → FAIL ❌

参考指標: 全員一致率 > 50%
  35.0% < 50%
  → FAIL ❌
```

**予想とのズレの考察:**
```
全指標が予想と逆方向になった

最大の驚き:
  旧データ（3bモデル）のCFR: 46〜51%
  新データ（7bモデル）のCFR: 14.3%
  差: 約35pp

解釈:
  グループシンクはモデルサイズに強く依存する
  3bモデル: 他ノードに引きずられやすい → CFR高い
  7bモデル: 独立した判断を保てる → CFR低い

全員一致率も大幅に異なる:
  3bモデル（v4）: 92%
  7bモデル（今回）: 35%
  → 7bモデルは意見が分かれやすい
  → グループシンクの前提（全員一致が多発）が崩れた
```

**想定外の観察:**
```
NCA-LLMが単一エージェントより精度が高かった
  NCA: 28.0% エラー率
  単一平均: 37.7% エラー率
  → NCA構造がエラー修正として機能している

これはCase Aとして事前に想定していた
「NCA構造はグループシンクではなく
 エラー修正をしている」

7bモデルでは:
  shared_premisesがなくても
  複数のノードが独立した判断を持てる
  → 多数決によってエラーが修正される
  → グループシンクではなく「多様性」が機能している
```
