## ① Feature Claim（機能の主張）

```
「7b異種NCAの多様性効果（CFR低下）は
 world_consistencyタスクだけでなく
 数学タスク（elementary・middle・high）でも
 観察される」
```

**Claimが曖昧でないか確認:**
- [x] 主語（何が）が明確か
      → 「7b異種NCA vs 7b同種NCA」
- [x] 述語（どう変わるか）が測定可能か
      → CFR（wrong_unanimous / total_unanimous）で測定
- [x] 対象（どのデータ・条件で）が特定されているか
      → 4タスクセット × 4条件
      → 全条件でv1〜v4と同じ設定
        （steps=3・roles=S/V/C・agree=[30,80,80]・Counter多数決）

---

## ② North Star Link（北極星との接続）

```
Claim成立
  → 多様性効果（CFR低下）が
    意味タスク・数学タスク問わず観察される
  → グループシンクとその抑制は
    タスク種別に依存しない普遍的な現象
  → 「shared_premisesの欠如による失敗」は
    特定ドメインに限らない
  → 北極星A-4
    「AIの失敗の多くは（= 幅広い領域で）
     shared_premisesが更新されなかった結果」
    の普遍性を支える証拠になる

Claim不成立（数学では効果なし）:
  → 「多様性効果はドメイン依存」として記録
  → どのドメインで機能するかのマッピングが進む
  → A-4の適用範囲の明確化につながる
```

**接続の論理:**
```
Claim成立
  → CFR(7b異種) < CFR(7b同種) が
    全4タスクセットで確認される
  → タスク種別に関わらず
    多様性がshared_premisesの代替として機能する
  → A-4の普遍性（「AIの失敗の多く」）を
    複数ドメインで実証
```

## ④ Success Criteria（成功基準）

| 指標 | ベースライン | 成功閾値 | 優先度 |
|------|------------|---------|--------|
| 7b異種 CFR < 7b同種 CFR（math_elem） | 未測定 | 方向が一致（3タスク全て） | 必須 |
| 7b異種 CFR < 7b同種 CFR（math_mid） | 未測定 | 方向が一致（3タスク全て） | 必須 |
| 7b異種 CFR < 7b同種 CFR（math_high） | 未測定 | 方向が一致（3タスク全て） | 必須 |
| p値（各タスク） | — | 参考値として記録 | 参考 |
| 3b異種 CFR ≈ 3b同種 CFR（全タスク） | 38.6%≈38.5% | 差なし | 参考 |

**判定ロジック:**
```
必須指標（3つ全て成立）→ FULL SUCCESS
  多様性効果は数学タスクでも普遍的

必須指標（一部成立）→ PARTIAL
  特定の難易度でのみ効果あり
  どの難易度で機能するかがわかる

必須指標（全て不成立）→ FAIL
  多様性効果はworld_consistencyのみ
  → A-4の適用範囲がworld_consistencyに限定
```

**注意: p値について:**
```
各タスクセットのn=75〜100では
検出力が不足する可能性がある

旧リポジトリの知見:
  math_middle CFR（v9d）: 27.4%（n=62）
  math_high CFR（v9d）:   21.1%（n=57）

7b異種でこれより低くなるか？
が主な観察点

p<0.05が達成できない場合も
「方向性」として記録する
```

| math_middle | 20〜35% | 10〜25% | -5〜15pp |
| math_high | 15〜30% | 8〜20% | -5〜15pp |

|------------|--------------|--------------|---------|
| math_elementary | 10〜25% | 10〜25% | ≈0pp（3bでは効果なし） |
| math_middle | 25〜40% | 25〜40% | ≈0pp |
| math_high | 20〜35% | 20〜35% | ≈0pp |

**予想の確信度:** 中

理由:
  旧リポジトリで数学タスクのCFRを測定済み
  ただし旧設定（異種・7b）と今回の条件が
  完全には一致しない

**予想が外れた場合に最初に疑うこと:**
```
Case A: 数学タスクでCFRが非常に低い（<5%）
  → 「7bモデルは数学が得意すぎて
    グループシンクが起きにくい」
  → タスクの難易度調整が必要

Case B: 3bでも数学タスクで多様性効果が出る
  → 「数学タスクは意味タスクより
    多様性が機能しやすい」
  → ドメイン特性の発見

Case C: 7b同種と7b異種でCFRが同じ（数学で）
  → 「多様性効果はworld_consistencyのみ」
  → A-4の適用範囲の限定
```
