  「7b同種split_acc(78.9%) > 7b異種split_acc(32.0%)」

**予想とのズレの考察:**
```
最大の発見:
  数学タスクでは7b異種のsplit精度が崩壊する
  math_middle: 7b異種 split_acc=32.0%（7b同種78.9%）
  = 意見が分かれた時に正解を選べない

タスク種別による構造の違い:
  world_consistency（知識・判断）:
    7b異種 split_acc=64.6% > 7b同種46.7%
    = 多様性が「生産的な不一致」を生む

  数学タスク（計算・推論）:
    7b異種 split_acc=32.0% < 7b同種78.9%
    = 多様性が「ノイズ」を生む
    = 異種モデルが意見を分けても
      正解を選べない
```

**想定外の観察:**
```
math_middleで7b異種のunanimitｙが
7b同種より高い（77.7% vs 66.1%）

= 異種モデルでも数学では全員一致しやすい
  かつ全員一致した時の誤り率が高い（CFR 16.1%）

= 「異なるモデルが同じ間違いに収束する」
  ではなく「正解が1つなので収束しやすい」
  というタスク構造の問題
```

**North Star A-4への接続:**
```
確認できたこと:
  多様性効果はタスク種別に依存する

  知識・判断タスク（world_consistency）:
    多様性 → 生産的な不一致 → 正解を選べる
    = shared_premisesの代替として機能

  計算タスク（数学）:
    多様性 → ノイズ → 正解を選べない
    = shared_premisesの問題ではなく
      計算能力そのものの限界

A-4の適用範囲の明確化:
  「知識・判断タスクでは
   shared_premisesの欠如が失敗の根本」
  「計算タスクでは
   能力の限界が支配的変数」

次のClaim（MVE-06）:
  論理推論タスク（A）の設計へ
  「論理規則を明示すると
   CFRは改善するか」
```
