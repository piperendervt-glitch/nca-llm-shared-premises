## ② North Star Link（北極星との接続）

```
Claim成立
  → Grok/ChatGPT生成タスクでも
    Claude生成タスクと同じCFRパターンが観察される
  → WCの+16.3ppは
    「Claudeのタスク設計バイアス」ではなく
    「架空ルール型タスクの本質的な性質」
  → LLM自作タスク問題が解消される
  → MVE-01〜09の探索フェーズの発見が
    外部妥当性を持つことが確認される
  → A-4の外部妥当性が確立される

Claim不成立の場合:
  → WCの結果はClaudeのタスク設計バイアス
  → 探索フェーズの発見を再設計する必要がある
  → A-4の検証方法を根本から見直す
  = これも重要な発見として記録する
```

**接続の論理:**
```
Claim成立
  → タスク作成者（Grok/ChatGPT）と
    タスク解答者（qwen2.5等）が分離された状態で
    同じCFRパターンが観察される
  → LLMエコシステムの循環バイアスが
    結果に影響していないことが確認される
  → 架空ルール型タスクでの多様性効果が
    タスク作成者に依存しない普遍的現象
  → A-4の実験的証拠の信頼性が確立される
```

## ④ Success Criteria（成功基準）

**事前宣言（north_star.mdで確定・変更禁止）:**

| 指標 | セットA（Grok） | セットB（ChatGPT） | 優先度 |
|------|--------------|-----------------|--------|
| CFR差 > +5pp | Grokタスクで確認 | ChatGPTタスクで確認 | 必須 |
| 正方向（7b_het < 7b_homo） | 必須 | 必須 | 必須 |
| Claude WCとの方向一致 | +16.3pp参照 | +16.3pp参照 | 参考 |

**判定ロジック:**
```
セットA・B両方でCFR差 > +5pp → FULL SUCCESS
  外部妥当性確立
  探索フェーズの発見が裏付けられた

どちらか一方のみ → PARTIAL
  タスク生成AIによって結果が異なる
  追加分析が必要

両方不成立 → FAIL
  WCの結果はClaudeのバイアスの可能性
  探索フェーズの再設計が必要
```

## ⑤ Prediction（事前予想）

| 指標 | セットA（Grok） | セットB（ChatGPT） | 根拠 |
|------|--------------|-----------------|------|
| 7b同種 CFR | 20〜35% | 20〜35% | Claude WC(30.6%)と同程度 |
| 7b異種 CFR | 5〜20% | 5〜20% | Claude WC(14.3%)と同程度 |
| CFR差 | +5〜25pp | +5〜25pp | Claude WC(+16.3pp)を参照 |
| split率(7b異種) | 30〜70% | 30〜70% | Claude WC(65%)を参照 |

**予想の確信度:** 中

理由:
  架空ルール型タスクの性質が
  タスク作成者に依存しないと仮定すれば
  同様のパターンが出ると予想
  ただしGrok/ChatGPTが作る
  架空ルールの難易度・多様性が
  Claudeと異なる可能性がある

**予想が外れた場合に最初に疑うこと:**
```
Case A: CFR差が小さい（<5pp）
  → Grok/ChatGPTが作るタスクが
    Claude WCより簡単すぎる
  → 天井効果が起きている可能性

Case B: CFR差が逆方向
  → 外部タスクの設計が
    Claude WCと根本的に異なる
  → 「架空ルール」の解釈が
    LLMによって異なる

Case C: セットA・Bで結果が大きく異なる
  → 「架空ルール型」というカテゴリが
    タスク作成AIによってばらつきが大きい
  → タスクの品質基準が必要
```
